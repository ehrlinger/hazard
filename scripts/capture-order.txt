# capture-order.txt
#
# Dependency-aware run order for the acceptance-corpus capture.
# Some .sas scripts write estimate data sets (under $HZEXAMPLES/sasest/
# via LIBNAME EXAMPLES) that later scripts read with PROC HAZPRED
# INHAZ=EXAMPLES.<dataset>.  A consumer run before its producer will
# fail with exit=16 (SAS: data set not found) or produce wrong output.
#
# FORMAT
#   - One .sas filename per line (basename only, no directory, no
#     .sas extension required — both "hm.death.AVC" and
#     "hm.death.AVC.sas" work).
#   - Blank lines ignored.
#   - Lines starting with '#' ignored.
#
# USAGE
#   The capture instructions' main loop reads this file and iterates
#   over it.  Anything listed here runs in the order given.  Any .sas
#   files present on-disk but NOT listed here are run AFTER the
#   manifest, in alphabetical order, as a best-effort catch-all.
#
# PROVENANCE
#   Order derived 2026-04-22 by grepping LIBNAME EXAMPLES / OUTHAZ /
#   INHAZ / DATA EXAMPLES.* statements across examples/*.sas.  The
#   producer→consumer pairs surfaced were:
#
#     hz.death.AVC           OUTHAZ=EXAMPLES.HZDEATH
#       └─ hp.death.AVC       INHAZ=EXAMPLES.HZDEATH
#
#     hm.death.AVC           DATA EXAMPLES.HMDEATH; SET OUTEST
#       ├─ hp.death.AVC.hm1   INHAZ=EXAMPLES.HMDEATH
#       ├─ hp.death.AVC.hm2   INHAZ=EXAMPLES.HMDEATH
#       └─ hs.death.AVC.hm1   INHAZ=EXAMPLES.HMDEATH
#
#     bs.death.AVC           DATA EXAMPLES.BSAVC{E,C,L}
#       └─ bs.death.AVC.summary SET EXAMPLES.BSAVC{E,C,L}
#
#   Other .sas examples either (a) write OUTHAZ=OUTEST to a purely-
#   local WORK library and don't publish anything consumed elsewhere,
#   or (b) read from external LIBNAMEs (CHSPTR94, TGASW, PTCAMHI,
#   CABGKUL, VAMEDCBG) that aren't populated by any script in this
#   corpus.  Those are run after the dependency-ordered block, order
#   among them doesn't matter.

hz.death.AVC

# ---- Consumers of HZDEATH (needs hz.death.AVC) ----
